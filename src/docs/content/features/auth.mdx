<br/>

Let's walk through the authentication system! Follow [Supabase Setup](/docs/tutorials/supabase) first to get your project set up.

Simply plug the auth methods you want into [`appConfig`](/docs/features/config) and the `/signup` and `/login` pages will
automatically adjust to your choices.

```config.js
const appConfig = {
    // Auth
    auth: {
        methods: ['magiclink', 'google'],
    }
};
```

<br />

You can also add the `protectedRoutes` key to specify which routes require authentication:

```config.js
const appConfig = {
    // Auth
    protectedRoutes: [
        '/dashboard',
        '/account'
    ]
};
```

<br />
<br />

## Config

<div id="password-based-auth" />

| Key               | Description                                                                                           |
|-------------------|-------------------------------------------------------------------------------------------------------|
| `authMethods`     | A list of authentication methods to enable (`password`, `magiclink`, `google`, `apple`, or `github`). |
| `protectedRoutes` | An array of routes that require authentication.                                                       |
| `afterLoginPath`  | The path to redirect to after a successful login.                                                     |
| `afterSignupPath` | The path to redirect to after a successful signup.                                                    |

<Info>
    The `afterLoginPath` and `afterSignupPath` keys are used to redirect users after they log in or sign up (no need to change them).
</Info>

That's it! You're all set up with a secure and ready-to-use authentication system.
