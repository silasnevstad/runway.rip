<div id="overview" />

1. Create a [Supabase](https://supabase.com/sign-up) account.

2. Create a [new project](http://database.new) and find your project `URL` and `anon public` API key.

3. Copy them into `.env.local`:

    ```.env.local
    NEXT_PUBLIC_SUPABASE_URL=https://<project-id>.supabase.co
    NEXT_PUBLIC_SUPABASE_ANON_KEY=eyXXX.eyXXX.C-UIPXXX
    ```

4. Head to [Email Templates](https://supabase.com/dashboard/project/_/auth/templates) and depending on the auth methods you're using,
update the appropriate templates:

    - If you are using magic link auth, update the `Magic Link` template.
    - If you are using email/password auth, update the `Confirm Signup` template.
    - If you are using social auth, no changes are needed.

    Update the &lt;a&gt; with the following href:

    ```plaintext
    <a
        href="{{ .SiteURL }}/auth/confirm?token_hash={{ .TokenHash }}&type=email&next={{ .RedirectTo }}"
    >
    ```

    <Tip>
        Customize these email templates to match your brand.
    </Tip>

5. Going to production? Update your [Site URL](https://supabase.com/dashboard/project/_/auth/url-configuration) to your production URL.
Then add `https://yourdomain.com/**` and `http://localhost:3000/**` to [Redirect URLs].

<div id="emails" />
## Emails

To send [Auth](/docs/features/auth) emails from your domain, you need to set up your [email provider](/docs/features/emails)
and update the [SMTP Settings](https://supabase.com/dashboard/project/_/auth/smtp).

<Callout>
    This will allow you to send magic links and other emails from your domain.
</Callout>
