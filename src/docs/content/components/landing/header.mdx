<br />

The `Header` component is a customizable header that can include a **logo**, **app name** (both pulled from [config.js](/docs/features/config)), **navigation links**, and a **call-to-action button**.
It is designed to be responsive and can adapt to different screen sizes automatically (dropdown menu on mobile).

<Callout>
    It uses [`TextLink`](/docs/components/text-link) and [`Button`](/docs/components/button) components for navigation and call-to-action respectively.
</Callout>

## Usage

The component automatically pulls the **logo** and **app name** from [config.js](/docs/features/config) if they are defined, so you can simply
import the component and use it like this:

```js
import Header from '@/components/organisms/Header';

<Header
    navLinks={[
        { title: 'Home', href: '/' },
        { title: 'About', href: '/about' },
        { title: 'Contact', href: '/contact' },
    ]}
    ctaButton={{
        label: 'Get Started',
        href: '/get-started',
    }}
/>
```

<br/>

Since `Header` uses [`TextLink`](/docs/components/text-link) and [`Button`](/docs/components/button) components, you can pass any prop that they accept to customize the header, for example:
```js
// For nav links
{
    title: 'Home',
    href: '/',
    props: { underline: true, fade: true },
}

// For CTA button
{
    label: 'Get Started',
    href: '/get-started',
    props: { size: 'lg', color: 'primary' },
}
```

## Props

| Name         | Type    | Default                         | Description                                                                                                                                                 |
|--------------|---------|---------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------|
| showLogo     | boolean | `true`                          | If true, the logo will be displayed.                                                                                                                        |
| showAppName  | boolean | `true`                          | If true, the app name will be displayed.                                                                                                                    |
| navLinks     | array   | `landingConfig.header.navLinks` | Array of navigation links. Each link is an object with `name`, `href`, and optional `props`. For more details, see [config](/docs/features/config#landing-page). |
| ctaButton    | object  | `null`                          | Call-to-action button. An object with `label`, `href`, and optional `props`.                                                                                |
| background   | string  | `"bg-bg-100 dark:bg-bg-800"`    | Background color (use [tailwind.config](/docs/features/config#color-theme)).                                                                                |
| bottomBorder | boolean | `false`                         | If true, a bottom border will be displayed.                                                                                                                 |
| sticky       | boolean | `false`                         | If true, the header will be sticky.                                                                                                                         |
| account      | boolean | `true`                          | If true, the account button will be displayed.                                                                                                              |
| className    | string  | `""`                            | Additional [Tailwind](https://tailwindcss.com) utility classes.                                                                                             |
| ...props     | object  | -                               | Any other valid HTML attributes for the <code>&lt;header&gt;</code> element.                                                                                |

<Info>
    The `sticky` prop is disabled by default, so the header will scroll with the page. To make it sticky, set the `sticky` prop to `true`.
</Info>

<Caution>
    The `account` prop can only be used under `/app/(auth)` pages since it uses the [`UserProvider`](/docs/features/auth).
    It uses the [`AccountCard`](/docs/components/auth/account-card) component.
</Caution>
